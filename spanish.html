Partyus

( Desarrollo de aplicaciones, diseño )

Partyus es una aplicación móvil para conectar a los organizadores de eventos con los clientes. Autoriza a los organizadores a crear un evento en el que los clientes pueden registrarse y pagar por la entrada si corresponde.
Una vez que tienes tu entrada, en la fecha del evento, el organizador escanea tu código QR y lo marca como unido al evento, evitando la falsificación de entradas.
Finalmente, el dinero de cada entrada se envía al organizador del evento a través de un backend complejo y seguro.



¿Cómo se desarrolló?
En primer lugar, tuve que investigar sobre diseño e interfaz de la aplicación para desarrollarla. A partir de la idea principal de conectar a los organizadores con los clientes, comencé a desglosarla hasta llegar a cada pantalla y lógica de la aplicación.

Para el frontend, se desarrolló en React Native, un framework para desarrollar aplicaciones móviles combinando HTML, javascript, React y CSS en un lenguaje llamado JSX.
Para el backend, lo desarrollé utilizando AWS Amplify. Utilicé 4 servicios principales en AWS como parte de Amplify:

  • Dynamo DB para la gestión de bases de datos con graphql

  • AWS Cognito e IAM para manejar los usuarios de la aplicacion y seguridad en los accesos

  • Lambda para crear funciones seguras que manejen pagos y creacion de reservas (datos sensibles)

  • S3 storage guardar imagenes del usuario y archivos del backend

Velpa

( Fullstack developer / fundador )

Funcionamiento
Velpa es una aplicacion para conectar a organizadores de experiencias al aire libre con gente que quiere ir a las mismas y manejar todo el sistema de reservaciones, mensajes.

La logica de la aplicacion es primero ofrecerle al organizador de la experiencia al aire libre la posibilidad de agregar un lugar si es que no esta en la aplicación. (Ej.: Monte everest) y es enviado a los admins para revisión.
Una vez que es aprovada, el guía puede crear todas las fechas que quiera con una gran personalización de su fecha de salida.

Finalmente es hora de que el cliente localize la aventura (Monte everest) y hacer la reserva si la fecha que anuncio el guia coincide con lo que el cliente quiere.

¿Como fue desarrollado?

Para comenzar, utilizé google maps para crear mapas personalizados y google OAuth para el inicio de sesion con google y una funcion que combina cuentas de IAM y google para evitar repeticiones.
Para los datos del usuario, utilizé dynamo DB para crear bases de datos no relacionales y con amplify datsatore para manejar un estado de datos en el cliente. Todo el sistema de informacion tiene accesos granulares dependiendo del usuario.
Finalmente, desarrolle una API REST para manejar todos los pagos y creacion de reservas de manera segura.

Pentesting

( Autoaprendizaje )

Objetivo:
Ser capaz de encontrar vulnerabilidades en distintos dispositivos y redes.

Herramientas/Tecnicas usadas:
• Kali linux
• Scripting
• SQL Injection
• SMB enumeration
• Man in the middle
• Exfiltración
• Fugas de información (OSINT)
• Pivoting
• Web enumeration
• Escalacion de Privilegios


Metodologia:
Primero, realizo un nmap o tecnicas de enumeracion al destino localizando todos los puertos abiertos y posibles vulnerabilidades. Si no hay vulnerabilidades conocidas abiertas, recurro a la ingeniería social o en busca de cualquier fuga de información. En caso de que sea un servidor web, Intentaré una inyección de sql para poder cargar un archivo de shell remoto.

Una vez dentro, ejecuto algunas técnicas para detectar mis privilegios y accedar a toda la información expuesta con mis permisos actuales. Si encuentro alguna vulnerabilidad para escalación de privilegios la utilizo, logrando asi mi acceso de root.

Finalmente, escaneo la red desde el cliente para buscar posibles puertos abiertos internos o servidores privados.

Administración de redes

( Proyecto escolar )

Objetivo:
En este proyecto implementé un sistema de administración de redes utilizando Ansible, Jump Server y Syslog. El sistema se utilizará para gestionar y monitorear una red de dispositivos, automatizar tareas administrativas de rutina y solucionar problemas de red.

Software utilizado:
• Ansible
• Jump Server
• Tacacs
• Syslog
• Dispositivos de red (routers, switches, firewalls, etc.)


Implementación:
La implementación implica configurar el nodo de control, el jump server y servidor Syslog, y configuración de los dispositivos de red para comunicarse entre sí configurados con ansible. El jump server se utiliza para proporcionar acceso remoto seguro a los dispositivos, con políticas de control de acceso implementadas para restringir el acceso a usuarios autorizados en un servidor TACACS. El servidor Syslog se usa para recopilar y analizar la red datos de tráfico, con alertas configuradas para detectar eventos de seguridad y comportamiento de red anormal.
