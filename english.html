Partyus

( App developement, Design )

Partyus is a mobile aplication for connecting events organizers with clients. It authorized organizers to create an event in where clients can register and pay for the ticket if it is the case.
Once you have your ticket, in the event date, the organizer scan your QR code and mark it as joined in the event avoiding ticket falsification.
Finally, the money of each ticket is sent to the event organizer through a complex and secure backend.

¿How was it developed?
First, I had to do research in design and app interface for developing it. From the main idea of connecting organizers with clients. I started to break down untill arriving to each screen and app logic.

For the frontend, it was developed in React Native, a framework for developing mobile apps by combining HTML, javascript, React and CSS in a language called JSX.
In the backend, I developed it using AWS Amplify. I used 4 main services in AWS part of Amplify:

  • Dynamo DB for database management with graphql

  • AWS Cognito and IAM for handling app users an security

  • Lambda for making secure cloud functions that handle payments

  • S3 storage for handling user images and data

Velpa

( Fullstack developer / founder )

Functionement
Velpa is a mobile aplication to help people that want to go hikking or in an open air adventure and doesn't know someone that guides them.

The app logic is first to give the adventure guides to propose an adventure if it doesn't exist in the app (Ej.: Mount everest) and it is send to admins for revision.
Once the adventure is aproved, the guide can request for the hability of adding his date of the hike with a lot of personalization.

Now it is time for the client to locate the adventure (Mount everest) and make a booking there if a guide published a date that fits the client's needs.

¿How was it developed?

First with google services, I handle google maps for a custom animated map and manage oAuth google sign in.
For the user information, all was developed in graphql combined with AWS Dynamo DB. It all manage a login and user authorization for accesing the data.
Finaly I used stripe and REST api's for secure managing payments and encrypted cards information.

Pentesting

( Self learning )

Objective:
Be able to detect vulnerabilities in a system and gain administrator control

Tools/Techniques used:
• Kali linux
• Scripting
• SQL Injection
• SMB enumeration
• Man in the middle
• Exfiltration
• Information leakage (OSINT)
• Pivoting
• Web enumeration
• Privilege escalation


Methodology:
First, I perform an nmap to the target locating all open ports and possible vulnerabilities. If no known vulnerabilities are open, then I fallback to social engineering or looking for any information exposure. In case it is a webserver, I will try an sql injection to be able to upload a remote shell file.

Once inside, I will run some techniques for detecting my permissions and lookout for all exposed information with my actual permissions. If I find any vulnerability for privesc,then use it for gaining root acces.

Finnaly, I scan the network in case there is some more devices and proceed with the exfiltration and cleanup.

Network administration

( School project )

Objective:
In this project, I had to implement a network administration system using Ansible, Jump Server, and Syslog. The system will be used to manage and monitor a network of devices, automate routine administrative tasks, and troubleshoot network issues.

Software used:
• Ansible
• Jump Server
• Tacacs
• Syslog
• Network devices (routers, switches, firewalls, etc.)


Implementation:
The implementation involve setting up the control node, the jump Server, and Syslog server, and configuring the network devices to communicate with these components with ansible prebuild code. The jump server will be used to provide secure remote access to devices, with access control policies in place to restrict access to authorized users in a TACACS server. The Syslog server will be used to collect and analyze network traffic data, with alerts configured for security events and abnormal network behavior.
