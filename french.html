Partyus

( Développement d'application )

Partyus est une application mobile qui permet de connecter les organisateurs d'événements avec les clients. Elle autorise les organisateurs à créer un événement où les clients peuvent s'inscrire et payer leur billet le cas échéant.
Une fois le billet acheté, le jour de l'événement, l'organisateur scanne le code QR du client et le marque comme présent pour éviter les fraudes de billets.
Enfin, l'argent de chaque billet est envoyé à l'organisateur de l'événement via une plateforme arrière complexe et sécurisée.

Comment cela a-t-il été développé ?
Tout d'abord, j'ai dû effectuer des recherches en design et en interface d'application pour le développement. À partir de l'idée principale de connecter les organisateurs avec les clients, j'ai commencé à décomposer chaque élément pour arriver à chaque écran et à la logique de l'application.

Pour le frontend, il a été développé en React Native, un framework pour le développement d'applications mobiles qui combine HTML, JavaScript, React et CSS dans un langage appelé JSX.
Pour le backend, je l'ai développé en utilisant AWS Amplify. J'ai utilisé 4 services principaux dans AWS faisant partie d'Amplify :

  • Dynamo DB pour la gestion de la base de données avec graphql

  • AWS Cognito et IAM pour gérer les utilisateurs de l'application et la sécurité

  • Lambda pour créer des fonctions cloud sécurisées qui gèrent les paiements

  • Stockage S3 pour la gestion des images et des données des utilisateurs

Velpa

( Fullstack développeur / fondateur )

Functionement
Velpa est une application mobile pour aider les personnes qui souhaitent faire de la randonnée ou des aventures en plein air et qui ne connaissent personne pour les guider.

La logique de l'application consiste d'abord à permettre aux guides d'aventure de proposer une aventure si elle n'existe pas encore dans l'application (par exemple : l'Everest) qui est ensuite envoyée aux administrateurs pour révision.
Une fois que l'aventure est approuvée, le guide peut demander la possibilité d'ajouter sa date de randonnée avec beaucoup de personnalisation.

Finalement, c'est au tour du client de localiser l'aventure (l'Everest) et de réserver une place si un guide a publié une date qui convient aux besoins du client.

Comment a-t-il été développé ?

D'abord, j'ai utilisé les services Google avec Google Maps pour une carte personnalisée et gérer la connexion Google oAuth.
Pour les informations utilisateur, tout a été développé en graphql combiné avec AWS Dynamo DB. Tout gère une connexion et une autorisation utilisateur pour accéder aux données.
Enfin, j'ai utilisé Stripe et des API REST pour gérer de manière sécurisée les paiements et les informations de carte cryptées.

Pentesting

( Auto-apprentissage )

Objectif:
Être capable de détecter les vulnérabilités dans un système et prendre le contrôle administratif.

Outils/Techniques utilisés :
• Kali Linux
• Scripting
• Injection SQL
• Énumération SMB
• Homme du milieu
• Exfiltration
• Fuites d'informations (OSINT)
• Pivotement
• Énumération Web
• Élévation de privilèges


Méthodologie :
Tout d'abord, je réalise un scan nmap sur la cible pour repérer tous les ports ouverts et les vulnérabilités potentielles. Si aucune vulnérabilité connue n'est détectée, alors je me tourne vers l'ingénierie sociale ou la recherche d'exposition d'informations. Si c'est un serveur web, j'essaierai une injection SQL pour pouvoir télécharger un fichier de shell distant.

Une fois à l'intérieur, j'utiliserai des techniques pour détecter mes autorisations et rechercher toutes les informations exposées avec mes autorisations actuelles. Si je trouve une vulnérabilité pour l'élévation de privilèges, je l'utiliserai pour obtenir un accès root.

Enfin, je scanne le réseau au cas où il y aurait d'autres dispositifs et je procède à l'exfiltration et au nettoyage.

Administration du réseaux

( Project scholaire )

Objectif :
Dans ce projet, j'ai dû mettre en place un système d'administration réseau en utilisant Ansible, Jump Server et Syslog. Le système sera utilisé pour gérer et surveiller un réseau de périphériques, automatiser les tâches administratives courantes et résoudre les problèmes de réseau.

Logiciels utilisés :
• Ansible
• Jump Server
• TACACS
• Syslog
• Périphériques réseau (routeurs, commutateurs, pare-feu, etc.)


Mise en œuvre :
La mise en œuvre consiste à configurer le nœud de contrôle, le jump server et le serveur Syslog, et à configurer les périphériques réseau pour communiquer avec ces composants avec du code préétabli dans Ansible. Le jump server sera utilisé pour fournir un accès distant sécurisé aux périphériques, avec des politiques de contrôle d'accès en place pour restreindre l'accès aux utilisateurs autorisés dans un serveur TACACS. Le serveur Syslog sera utilisé pour collecter et analyser les données de trafic réseau, avec des alertes configurées pour les événements de sécurité et les comportements anormaux du réseau.
